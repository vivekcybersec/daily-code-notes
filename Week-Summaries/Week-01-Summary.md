# Week 1 Cybersecurity + Python Foundation — Practical Learning Summary

## Overview

This week was focused on rebuilding technical execution discipline and connecting Python basics with real cybersecurity thinking. The main objective was not mastering tools, but building the habit of solving security-style problems using code and analytical reasoning.

The focus areas were:

* Python execution for security data handling
* Basic log and IOC processing logic
* Introductory CTI analytical thinking
* Networking observation from defender perspective
* Building daily output consistency

This week was intentionally built around simple but practical tasks instead of theoretical learning.

By the end of the week, Python was used for:

## Networking Mindset Shift (Very Important)

This week was not focused on offensive networking.
The focus was defender observation and anomaly awareness.

### Skills Developed

* Interface and port awareness
* Understanding normal vs abnormal behaviour patterns
* DNS traffic behaviour basics
* Failed login pattern recognition logic
* Suspicious behaviour scenario thinking

### Key Mental Shift

Instead of asking:
“How to attack?”

Started asking:
“What would look abnormal in logs or traffic?”

This is critical for SOC and CTI roles.

Security Events → Suspicious Conditions → Risk Score → Alert Output

This is the foundation of:

* SOC tooling
* Detection engineering
* CTI automation
* AI-assisted threat detection (future phase)
